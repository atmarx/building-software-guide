*[SBOM]: Software Bill of Materials - a complete inventory of components in software
*[CVE]: Common Vulnerabilities and Exposures - standardized identifier for security vulnerabilities
*[NVD]: National Vulnerability Database - NIST's database of vulnerability information
*[SCA]: Software Composition Analysis - tools that identify components and vulnerabilities in code
*[SLSA]: Supply-chain Levels for Software Artifacts - framework for supply chain integrity (pronounced "salsa")
*[CI]: Continuous Integration - automated building and testing on every code change
*[CD]: Continuous Deployment - automated deployment of code changes to production
*[CI/CD]: Continuous Integration and Continuous Deployment - automated build, test, and deploy pipelines
*[semver]: Semantic Versioning - versioning scheme using MAJOR.MINOR.PATCH format
*[transitive dependency]: A dependency of your dependency - packages you didn't install directly
*[lock file]: A file recording exact resolved versions of all dependencies
*[vendoring]: Copying dependency source code directly into your project
*[bus factor]: The number of people who would need to be unavailable before a project fails
*[typosquatting]: Registering package names similar to popular packages to trick users
*[dependency confusion]: Attack exploiting package managers that check public registries for internal package names
*[RCE]: Remote Code Execution - a vulnerability allowing attackers to run arbitrary code
*[CVSS]: Common Vulnerability Scoring System - standardized severity ratings for vulnerabilities
*[OpenSSF]: Open Source Security Foundation - cross-industry collaboration for open source security
*[CISA]: Cybersecurity and Infrastructure Security Agency - US federal agency for cybersecurity
*[OSI]: Open Source Initiative - organization that reviews and approves licenses as "open source"
*[copyleft]: A licensing approach requiring derivative works to use the same license
*[permissive license]: A license that allows use with minimal restrictions
*[PHI]: Protected Health Information - health data covered by HIPAA regulations
*[PII]: Personally Identifiable Information - data that can identify an individual
*[HIPAA]: Health Insurance Portability and Accountability Act - US law governing health data privacy
*[medallion architecture]: Data lake pattern with bronze (raw), silver (cleaned), and gold (refined) layers
*[data lake]: A storage repository holding large amounts of raw data in native format
*[IaC]: Infrastructure as Code - managing infrastructure through version-controlled configuration files
